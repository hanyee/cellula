Cellula

Cellula.js 提供了一种web开发的思路,即是讲web页面应用片段化,模块化,颗粒化.并提供了 cell 用于表示模块或片段的最小集, 提供 element 用于为各种数据结构建立模型, 提供 collection 来管理cell和element. 提供 message 来实现模块和片段间的通信, 提供 class 作为面向对象封装的核心并, 以上cell, element, collection 和 message 都是基于 Class 这个强大的核心构建的, 提供 util 来弥补对 dom 框架依赖的缺憾. 在 Cellula 的基础上你可以任意发挥你的架构思路, 玩转各种看似简单时而又复杂的 web 页面开发.

Cellula 目前托管在 GitHub , 并且采用 MIT License 授权, 这里有尚不完善的 test case.

Introduction

通常前端开发人员在开发 web 页面或者 webapp 时，为了迅速或后续的通用性会用到或者自己写很多的‘插件’(widget), 但你也经常会发现在你想用已有的插件时，它并不像你想象的那般完美，要么挑来挑去一直没有心仪的， 要么看到一个差不多的，但还是要看说明或者源码，再改造成真正想要的插件，或者干脆自己动手实现一个， 但是你实现的这个，后续对于自己或者其他的开发人员可能还是会遇到同样的遭遇。往往这种情况下，我们发现 写插件的时候所谓的“硬编码，死关联”很多，我们会抱怨这个插件不够颗粒化，抱怨配置和改造不是那么容易， 但同时我们自己在开发的时候也懒得去更进一步考虑和规划这个事情，因为你这样做了会让你觉得花费了很多不必要的精力。

Cellula.js 便可以很好的解决这个问题，它提供了一种web开发的思路,即是将web页面应用片段化,模块化,颗粒化. 在 Cellula 的基础上再开发插件，会帮助你更容易的组织插件的每个“颗粒”，更容易建立对象和模型的概念。 Cellula 围绕自带的 Class 构造， Class 为 Javascript 提供了良好的面向对象的特性， 并集成了接口调用，事件等特性，这样就可以使得由 Class 构造出的类型或类型的实例都有相同的特性，达到表现统一。 同时 Cellula 提供了 cell 用于表示模块或片段的最小集, 提供 element 用于为各种数据结构建立模型, 提供 collection 来管理cell和element. 提供 message 来实现模块和片段间的通信, 提供 util 来弥补对dom框架依赖的缺憾. 在 Cellula 的基础上你可以任意发挥你的架构思路,开发类似于 Backbone.js 或者 Spine.js 的各种 MVC 小框架，玩转各种看似简单时而又复杂的web页面开发.

Cellula.Class

Class 是一个类构造器(Class type constructor)，作为javascript在oop设计和编码时的补充。 Class做为Cellula的基础，提供稳定且强大的功能，内建了事件模型和接口方法调用模型。 应用Class你可以很轻松的创建一个“class”类型，并保持良好的oo的特性。由Class创建的“类”仍然是个构造器(constructor)。 例： 

var SomeClass = new Cellula.Class('SomeClass' ,{
    init:function(arg){
        alert('i am called when an instance is creating by SomeClass!')
        alert('hello' + arg);
    },
    prop:"some prop"
});

var classInstance = new SomeClass('world!');
alert(classInstance.prop === "some prop");

inheritsSubClass.inherits(SomeClass)
inherits 是 Class 提供的继承方法，任何一个由 Class 创建的类都可以继承和被继承，例： 

var SubClass = new Cellula.Class('SubClass' ,{
    init:function(arg){
        this._super(arg);
        alert('i am a SubClass inherited from SomeClass!')
    },
    someprop:"someprop"
}).inherits(SomeClass);

var subClassInstance = new SubClass('world!');
alert(subClassInstance.prop === "some prop");
alert(subClassInstance.someprop === "someprop");

上面的例子中 Class 创建了一个新的类 SubClass ，同时 SubClass 继承了前面创建的类 SomeClass，并拥有了父类 SomeClass 的属性 prop, 同时重写了父类的 init 方法，替代父类的 init。这里用到了 this._super 这个方法用于调用父类的同名方法，是 Class 提供的一种方便的语法糖。

这里需要注意 inherits 是由 Class 创建的类的自有属性方法，如同 Function 的 call，length 等， 与其 prototype 的属性是不同的，下面介绍的 registerInterface， applyInterface, removeInterface 等方法是写在 prototype 上的。所以请注意区别。
